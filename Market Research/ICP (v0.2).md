# ICP & Positioning (v0.2) — MapWeld

Date: 2026-03-13

Delta vs v0.1: clearer split between (1) “drawing-first weld mapping” buyers and (2) “MES/platform” buyers.

---

## Primary ICP (best-fit wedge buyers)
### Segment
- **Pipe spool fabricators** and **offshore module yards** doing **high weld volume** where the input artifact is still **PDF isometrics** (not full BIM).

### Trigger conditions (what makes them buy)
- Turnover packs / MDRs create end-of-job fire drills
- Rework/repairs break traceability and cause NCRs
- Multiple welders/inspectors + frequent drawing revisions
- Poor connectivity / chaotic shopfloor (needs tablet-first, fast UI)

### Buyer + champion
- **Champion:** QC inspector / welding coordinator (feels the daily pain)
- **Economic buyer:** QA/QC Manager / Project Manager / Construction Manager
- **Influencers:** document control, client QA, NDT vendor

---

## Secondary ICP (where MapWeld is a ‘layer’)
- Shops that already have **ERP/MES/spool tracking** (PipeCloud-like) but still do weld mapping + evidence capture in PDFs/Excel.

Position as: **capture + structure + export layer** (not rip-and-replace).

---

## Positioning (tightened)
**Core line:**
> “Turn isometric PDFs into a live weld map on a tablet — in minutes — and export a turnover-ready register without retyping.”

**Contrast line (handles competitor split):**
- vs *free markup tools*: “not just markup — **structured weld data + pack outputs**.”
- vs *enterprise platforms*: “not a monolith — **fast wedge**, integrates via export.”

---

## Proof points to collect (for sales + website)
- Time to mark up 1 iso (minutes)
- % fewer missing weld IDs / missing NDT attachments
- Time saved during turnover week (hours / spool / module)
- Repair loop handling: can the weld ID lineage be shown in 30 seconds?
