# Pilot Onboarding Runbook

## Objective

Onboard each pilot team to day-1 activation in one session and assign clear owners for week-1 adoption.

## Roles

- MapWeld lead: runs kickoff and captures blockers.
- Pilot champion: accountable for activation completion.
- Backup champion: covers absences and report handoff.

## Kickoff agenda (45 minutes)

1. Scope alignment (5 min)
2. Workflow demo on sample (10 min)
3. Team setup on live pilot project (20 min)
4. Activation checkpoint and owners (10 min)

## Required outcomes by end of kickoff

- Pilot project selected and named.
- PDF loaded in app.
- First 10 weld records created.
- `.weldproject` saved and reopened.
- First export generated and reviewed.
- Week-1 review meeting scheduled.

## Blocker handling

- P0 (activation blocked): resolve within 24 hours.
- P1 (workflow friction): add workaround and resolve in weekly cycle.
- P2 (nice-to-have): capture in improvements backlog.

## Week-1 follow-through

- Day 2: async check-in message.
- Day 4: quick usage pulse (sessions, records, export count).
- Day 7: weekly review using feedback template.

## Exit from onboarding

Team exits onboarding phase when:

- activation checklist is fully complete;
- at least 2 users performed workflow steps;
- first weekly review is completed with action items.
