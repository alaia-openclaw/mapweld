# Outreach Sprint 1 Pack

Use this pack for the first 2-week outreach cycle.

## Outreach target

- 100 targeted contacts.
- 20+ meaningful responses.
- 8+ discovery calls.

## Target roles

- QA/QC manager
- Welding engineer/coordinator
- Document control lead
- Fabrication project manager

## Message variant A (QA/QC manager)

Subject: reducing weld handover rework

"Hi {{firstName}}, quick one.  
Are your teams still reconciling weld status between isometric PDFs and spreadsheets?

We built MapWeld to keep traceability on the drawing and export to Excel only when office needs it.  
No install, no account, no backend setup.

We are running a small 30-day pilot with mixed teams (shop + QA + office).  
Open to a 20-minute fit check this week?"

## Message variant B (welding engineer/coordinator)

Subject: traceability on the isometric

"Hi {{firstName}},  
I am speaking with teams that still update weld logs in separate sheets after field notes.

MapWeld lets you place welds directly on the PDF isometric, save a portable project file, and export a register for reporting.

Would you be open to a short call to see if a one-project pilot fits?"

## Message variant C (project manager/document control)

Subject: cleaner weld register handover

"Hi {{firstName}},  
Most register issues we see come from data living in too many places.

MapWeld keeps weld updates on the drawing and provides export-ready output without a heavy rollout.

If useful, I can share a 60-second walkthrough and pilot scope."

If weld traceability is already solved on your side, no worries.

If it is still spreadsheet-heavy, happy to share a short demo and see if a pilot makes sense."

## Follow-up #2 (7 business days)

"Last note from me, {{firstName}}.

Either way, appreciate the time."

## Community post template

"Weld traceability usually breaks at handover because updates happen off-drawing.

We are testing a lightweight approach:
- track directly on PDF isometrics
- keep one portable project file
- export register output when needed

Looking for 4-6 pilot teams (shop + QA + office) to test on one live project."

## Daily operating rhythm

- 30 new outreaches/day.
- 15 follow-ups/day.
- 20 minutes end-of-day tracker cleanup.

## Qualification handoff rule

Move lead to discovery call only when:

- active project exists;
- champion is identified;
- weekly review cadence is possible.
