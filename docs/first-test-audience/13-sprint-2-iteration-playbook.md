# Sprint 2 Iteration Playbook

Use this after first responses and first pilot check-ins to improve conversion and activation.

## Inputs to review

- Outreach tracker response patterns.
- Qualification call notes.
- Activation monitor failures.
- Weekly feedback severity trends.

## Decision framework

For each blocker, classify:

- Message problem (wrong audience or value framing).
- Process problem (pilot steps unclear).
- Product problem (feature gap or usability issue).

## Required sprint-2 updates

- Keep one winning outreach variant.
- Replace one underperforming outreach variant.
- Add one objection handling line to call script.
- Tighten activation checklist with top 2 recurring misses.
- Update pilot one-pager with clarified boundaries.

## Sprint-2 outreach target

- 150 new targeted contacts.
- 25+ meaningful responses.
- 10+ discovery calls.
- 2-3 additional pilot starts.

## Weekly review template

| Area | Current signal | Action | Owner | Due date |
|------|----------------|--------|-------|----------|
| Message fit | - | - | - | - |
| Qualification | - | - | - | - |
| Activation | - | - | - | - |
| Retention | - | - | - | - |

## Stop/continue rules

- Stop a channel if response quality stays low for 2 cycles.
- Continue channels with clear qualified replies.
- Escalate product blockers with severity >=4 immediately.
