# MapWeld First Pilot One-Pager

## What this pilot solves

Most teams track weld progress in a PDF plus a separate spreadsheet. That creates rework and late-stage corrections.  
MapWeld keeps weld data attached to the drawing and exports a register when office reporting is needed.

## What is included in this pilot

- Browser workflow on PDF isometrics.
- Weld/spool/part marker workflow in one workspace.
- Save/load `.weldproject` files for portability.
- Excel weld register export.
- Project health checks for missing links and common consistency issues.

## What is not included

- Multi-user cloud collaboration.
- Server-hosted company database.
- User authentication and permission tiers.
- Full databook automation as baseline pilot scope.

## Pilot setup

- Duration: 30 days.
- Scope: one active project.
- Team: 1 champion + 2-5 users across shop, QA/QC, office.
- Cadence: weekly 20-minute check-in.

## Pilot success metrics

- Day-1 activation completed (PDF loaded, 10+ welds, save and export once).
- At least 2 repeat sessions per week by the team.
- At least one handover/report cycle completed from exported data.
- Team confirms reduced manual reconciliation effort.

## Inputs required from pilot team

- One anonymized project PDF.
- Existing register template or expected report columns.
- Named project champion and backup contact.
- Agreement to share anonymized outcomes.

## Commercial path after pilot

- If pilot KPIs are met, convert to paid rollout or extended beta based on scope.
- If not met, run a focused 2-week remediation with agreed acceptance criteria.
